Discuss the scope of var, let, and const

var scope

var are global scope or functional scope when we declare var variabel that't hoisted in global scope that's why we can access under the functional scope or block scope.if var variabel declared in block scope that is accessible outside the block. we can re-assign the value of var variable.
here is the Example.
var greeter = "hey how";
function newFunction() {
var hello = greeter +" are you"; }

let scope

let are a block scope it accessible under the block scope.we can re-assign value under the block scope.let are not access outside the block.
here is the Example.
let greeting = "say Hi";
let times = 4;
if (times > 3) {
let hello = "say Hello";
console.log(hello);// "say Hello"
}
console.log(hello) // hello is not defined

Const scope

const are block scope. It is access under the block scope.const are constant variable its value never re-assign.
here is the Example.
const greeting = "say Hi";
const times = 4;
if (times > 3) {
const hello = "say Hello";
console.log(hello);// "say Hello"
}
console.log(hello) // hello is not defined

Tell us the use cases of null and undefined

In JavaScript null and undefined indicates the absense of value. null and undefined is a keyword in JavaScript.

undefined:

  • we declare a variabel and can't assign a value.when we use the variable thats is undefined
  • set value to undefined we get undefined
  • when need to return in a function but we can't return that's we get undefined
  • when need parameter pass but we can't pass parameter that't we get undefined
    • Example use cases:
  • when property doesn't exists that's why we get undefined
    const person = {}
    console.log(person.name);//undefined
  • when array index not in array that's why we get undefined
    const num=[0,1,2]
    console.log(num[3]);//undefined
